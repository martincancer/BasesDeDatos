Sentencias Básicas
DML en SQL

En esta clase vemos las cuatro sentencias DML más usadas en SQL Server: SELECT, INSERT, UPDATE y DELETE. La idea es entender para qué sirve cada una, cómo se escribe y en qué errores conviene no caer.

SELECT: consultar datos

SELECT se usa para leer datos de una tabla. No modifica registros: su trabajo es mostrar información para que podamos analizarla, filtrarla o usarla como base para otra decisión.

Es la sentencia más usada en SQL. Cada vez que querés ver qué hay cargado, revisar un conjunto de filas o comprobar qué registros cumplen una condición, seguramente vas a usar SELECT.
01

INSERT

Agrega filas nuevas a una tabla.

02

UPDATE

Modifica datos que ya existen.

03

DELETE

Elimina filas existentes con mucho cuidado.

Sintaxis básica
SELECT columna1, columna2
FROM nombre_tabla
WHERE condicion;

Si querés traer todas las columnas, se puede usar *, aunque no siempre es la mejor opción.

SELECT *
FROM Peliculas;
Ejemplos

Ver todos los registros

SELECT *
FROM Peliculas;

Sirve para revisar rápidamente todo el contenido de la tabla.

Filtrar por una condición

SELECT Titulo, Anio
FROM Peliculas
WHERE Anio >= 2020;

Acá solo aparecen las películas estrenadas desde 2020.

Una buena costumbre es pedir solo las columnas que necesitás. SELECT * es cómodo para practicar, pero en tablas grandes puede traer datos de más y afectar la performance de la consulta.

INSERT: agregar datos

INSERT se usa para cargar nuevas filas en una tabla. Cada vez que se registra un alumno, una película, una venta o un cliente nuevo, lo que estás haciendo es un INSERT.

Lo más recomendable es escribir siempre el listado de columnas. Eso vuelve la instrucción más clara y evita errores si el orden interno de la tabla cambia o si alguna columna no se completa manualmente.
Sintaxis básica
INSERT INTO nombre_tabla (columna1, columna2, columna3)
VALUES (valor1, valor2, valor3);
Ejemplos
  • 1
    Agregar una fila en la tabla Pelicula 
    INSERT INTO Peliculas (Titulo, Anio, Genero)
VALUES ('El secreto de sus ojos', 2009, 'Drama');
  • 2
    Agregar una fila en la tabla Directores 
    INSERT INTO Directores (Nombre, Pais)
VALUES ('Juan José Campanella', 'Argentina');
El orden de los valores debe coincidir exactamente con el orden de las columnas que declaraste. Si no, los datos pueden quedar mal cargados aunque la instrucción se ejecute.

UPDATE: modificar datos

UPDATE cambia valores de filas que ya existen. Es muy útil para corregir datos, actualizar estados o ajustar información que se guardó antes.

UPDATE no crea filas nuevas ni elimina filas. Lo que hace es tomar registros existentes y cambiar uno o más campos en ellos.
Sintaxis básica
UPDATE nombre_tabla
SET columna1 = valor1,
    columna2 = valor2
WHERE condicion;
Ejemplos

Actualizar un dato puntual

UPDATE Peliculas
SET Genero = 'Thriller'
WHERE PeliculaID = 4;

Solo cambia la fila cuya clave sea 4.

Antes de actualizar

SELECT *
FROM Peliculas
WHERE PeliculaID = 4;

Conviene revisar antes con un SELECT qué fila va a ser afectada.

Si omitís el WHERE, se actualizan todas las filas de la tabla. Ese es uno de los errores más comunes y más peligrosos al empezar.

DELETE: eliminar filas

DELETE elimina filas existentes de una tabla. Se usa cuando un registro ya no debe seguir estando en la base, pero justamente por eso es una sentencia que exige mucha atención.

Si UPDATE ya es delicado, DELETE todavía más. Antes de borrar, conviene comprobar con un SELECT exactamente qué filas cumplen la condición.
Sintaxis básica
DELETE FROM nombre_tabla
WHERE condicion;
Ejemplos
  • 1
    Borrar una fila puntual 
    DELETE FROM Directores
WHERE DirectorID = 7;
  • 2
    Comprobar antes de borrar 
    SELECT *
FROM Directores
WHERE DirectorID = 7;

Atención con este caso

DELETE FROM Directores;

Sin WHERE, esa instrucción elimina todas las filas de la tabla. La estructura de la tabla queda, pero los datos desaparecen.

SELECT Consultar o leer datos No traer columnas innecesarias si no hacen falta
INSERT Agregar filas nuevas Respetar el orden y tipo de los valores
UPDATE Modificar filas existentes No olvidar el WHERE
