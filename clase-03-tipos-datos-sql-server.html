Tipos de Datos
en SQL Server

Los tipos de datos le indican a SQL Server qué clase de valor puede guardar cada columna. Elegirlos bien ayuda a diseñar tablas más claras, eficientes y coherentes.

Panorama General

Los tipos de datos más comunes en SQL Server pueden agruparse en familias: numéricos, fecha y hora, cadenas de caracteres, booleanos y binarios.

Cada columna debería usar el tipo que mejor representa el dato real que va a guardar. No es lo mismo un precio, una fecha de nacimiento, un nombre o un archivo PDF.

Tipos Numéricos

Se dividen en exactos y aproximados. Los exactos guardan el valor tal cual; los aproximados se usan cuando puede haber pequeñas variaciones por cálculo.

Exactos
  • INT, SMALLINT, TINYINT, BIGINT: almacenan números enteros.
  • DECIMAL y NUMERIC: almacenan números con precisión y escala fija.
  • DECIMAL y NUMERIC son sinónimos en SQL Server.
Aproximados
  • FLOAT: almacena números de punto flotante.
  • Se usa mucho en cálculos científicos o mediciones.
  • No conviene para importes donde la exactitud es importante.
TipoQué guardaEjemplo
INTEnterosCantidad de alumnos, número de legajo
DECIMAL(10,2)Números con decimales exactos, entre parentesis se indica la longitud total del número a guardar y la cantidad de decimalesPrecios, saldos, promedios
FLOATDecimales aproximadosMediciones y cálculos técnicos
Para dinero o valores sensibles suele ser mejor usar DECIMAL antes que FLOAT.

Fecha y Hora

SQL Server ofrece varios tipos según se quiera guardar solo la fecha, solo la hora o una combinación de ambas con distinta precisión.

Tipos principales
  • 1
    DATE: guarda solo fecha.
  • 2
    TIME: guarda solo hora.
  • 3
    DATETIME: combina fecha y hora.
  • 4
    DATETIME2: igual que DATETIME, pero con mayor precisión.
  • 5
    DATETIMEOFFSET: fecha y hora con desplazamiento de zona horaria.
TipoUso comúnEjemplo
DATEFechas de nacimiento, inscripciones2026-04-09
TIMEHorarios de cursada18:30:00
DATETIME / DATETIME2Registro completo de fecha y hora2026-04-09 18:30:00
DATETIMEOFFSETEventos con zona horaria2026-04-09 18:30:00 -03:00
En diseños actuales, DATETIME2 suele ser una opción más completa que DATETIME.

Cadenas de Caracteres

Estos tipos sirven para guardar texto. La diferencia principal es si la longitud es fija o variable, y si admite o no Unicode.

Sin Unicode
  • CHAR: longitud fija.
  • VARCHAR: longitud variable.
  • CHAR rellena espacios si el texto es más corto.
Con Unicode
  • NCHAR: longitud fija con Unicode.
  • NVARCHAR: longitud variable con Unicode.
  • La N permite guardar caracteres Unicode.
La diferencia entre los tipos con N y los que no la tienen es si pueden almacenar o no caracteres Unicode. También cambia el espacio de almacenamiento que utilizan.
TipoLongitudEjemplo de uso
CHAR(10)FijaCódigos de longitud estable
VARCHAR(100)VariableCorreos, apellidos, direcciones cortas
NCHAR(10)Fija con UnicodeEtiquetas con soporte Unicode
NVARCHAR(200)Variable con UnicodeNombres y textos con acentos
Para textos generales en español, NVARCHAR suele ser una opción segura y flexible.

BIT y Tipos Binarios

Además de texto, fechas y números, a veces necesitamos guardar valores lógicos o datos binarios como archivos.

Tipo booleano

BIT almacena 1 para verdadero o 0 para falso.

Es ideal para campos como Activo, Aprobado, Pagado o TieneBeca.
BINARY
  • Datos binarios de longitud fija.
  • Hay que indicar el tamaño máximo al definir la columna.
VARBINARY
  • Datos binarios de longitud variable.
  • Muy útil cuando el tamaño puede cambiar.
Los tipos BINARY y VARBINARY son útiles para almacenar datos binarios como imágenes, documentos, PDFs u otros archivos.
TipoQué almacenaEjemplo
BITValor lógico simpleAlumno activo / inactivo
BINARYDatos binarios de tamaño fijoBloques de bytes de tamaño constante
VARBINARYDatos binarios de tamaño variableArchivos, imágenes y documentos
