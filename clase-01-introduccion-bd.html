Primera parte del recorrido: concepto de base de datos, tipos de bases de datos y explicación del SGBD como software que organiza, protege y administra la información.

¿Qué es una Base de Datos?

Una base de datos es una colección de datos relacionados entre sí, con un sentido específico, que representan algún aspecto del mundo real y están organizados para ser consultados y actualizados de forma eficiente.

La palabra clave es organización. No alcanza con guardar datos: los datos tienen que poder encontrarse, relacionarse y mantenerse consistentes con el tiempo. Una base de datos bien diseñada hace posible todo eso.

El problema que resuelven

Antes de las bases de datos, la información se guardaba en archivos planos, planillas u hojas de cálculo. A medida que los datos crecen, esos formatos empiezan a fallar:

Sin base de datos
  • Datos repetidos en múltiples archivos
  • Sin control: cualquiera edita cualquier cosa
  • Inconsistencias entre registros: "Juan Pérez" y "Juan Perez" como personas distintas
  • Búsquedas lentas
  • Riesgo de pérdida de información
Con base de datos
  • Dato único, almacenado una sola vez
  • Control de acceso por usuarios y permisos
  • Integridad garantizada por reglas
  • Consultas más rápidas y ordenadas
  • Transacciones y backups para recuperación
Ejemplos cotidianos de bases de datos: el sistema de alumnos de un colegio, el inventario de un almacén, las cuentas de un banco, el catálogo de una biblioteca digital o una app de turnos médicos.

Tipos de Bases de Datos

Existen distintos modelos para organizar y almacenar datos. Los dos grandes grupos hoy en día son las bases de datos relacionales (SQL) y las no relacionales (NoSQL).

Relacionales (SQL)
  • Diseñadas en los años 70. El modelo más maduro.
  • Datos organizados en tablas con filas y columnas.
  • Las tablas se relacionan entre sí mediante claves.
  • Escalan verticalmente (más potencia en un solo servidor (CPU, RAM)
  • Garantizan integridad ACID.
  • Se consultan con SQL (Lenguaje de Consulta Estructurado).
  • Ideales para sistemas financieros, ERP, CRM.
  • Ejemplos: SQL Server, MySQL, PostgreSQL, Oracle.
No relacionales (NoSQL)
  • Surgieron con la explosión de aplicaciones web complejas.
  • Modelos flexibles: documentos, clave-valor, grafos y columnas
  • No requieren un esquema fijo: cada registro puede tener estructura distinta.
  • Escalan horizontalmente (más servidores, no más potencia)
  • Sacrifican integridad en pos de la velocidad.
  • Se usan mucho en big data, redes sociales e IoT
  • Ejemplos: MongoDB, Redis, Cassandra, DynamoDB

Referencia rápida: qué es ACID

¿Qué es ACID?

ACID (Atomicidad, Consistencia, Aislamiento y Durabilidad) es una forma de asegurar que una transacción sea confiable. Dicho fácil: una operación importante se guarda completa y bien, o directamente no se guarda.

  • A - Atomicidad (Atomicity): todo o nada. Si una parte falla, toda la operación se revierte y nada se aplica.
  • C - Consistencia (Consistency): los datos deben seguir siendo correctos. Garantiza que la base de datos pase de un estado válido a otro, cumpliendo todas las reglas y restricciones
  • I - Aislamiento (Isolation): garantiza que las transacciones simultáneas no interfieran entre sí, evitando lecturas de datos erroneas.
  • D - Durabilidad (Durability): una vez guardado, el cambio no se pierde.

¿Qué es una transacción?

Una transacción en una base de datos es una unidad lógica de trabajo que agrupa una o varias operaciones, como leer, insertar, actualizar o borrar datos. Garantiza que todas las acciones se ejecuten exitosamente o ninguna lo haga.

Ejemplo: una transferencia bancaria de $100 de la cuenta A a la cuenta B implica dos acciones: reducir $100 en A y aumentar $100 en B. Si el sistema falla después de reducir en A pero antes de aumentar en B, la transacción se revierte para evitar la pérdida de dinero.

Sistema Gestor de Bases de Datos (SGBD)

Un SGBD (o DBMS, por sus siglas en inglés) es el software intermediario entre el usuario y los datos. Nadie accede directamente a los archivos en disco: todo pasa por el SGBD, que controla, organiza y protege la información.

Pensalo como el encargado de un depósito. Vos pedís una caja, él la busca, te la trae, registra quién la pidió, controla que la devuelvas bien, y se asegura de que dos personas no agarren la misma caja al mismo tiempo. Eso hace el SGBD con los datos.
  • 1
    Definición de datos: permite crear y modificar la estructura (tablas, columnas, tipos de dato, restricciones) mediante el lenguaje DDL (Data Definition Language).
  • 2
    Manipulación de datos: permite insertar, consultar, modificar y eliminar datos mediante el lenguaje DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE.
  • 3
    Control de acceso: gestiona usuarios, roles y permisos. Define quién puede leer, escribir o administrar cada parte de la base de datos.
  • 4
    Integridad: aplica reglas (claves primarias, foráneas, restricciones) para que los datos sean siempre correctos y consistentes.
  • 5
    Concurrencia: permite que múltiples usuarios trabajen al mismo tiempo sin que se pisen entre sí (bloqueos y transacciones).
  • 6
    Recuperación: ante una falla (corte de luz, error de hardware), el SGBD puede restaurar la base de datos a un estado consistente usando logs de transacciones y backups.

En esta materia usamos SQL Server. La herramienta visual más habitual es SSMS (SQL Server Management Studio).
SQL Server Management Studio (SSMS) es el programa que usamos para trabajar con SQL Server de forma visual. Desde ahí podemos conectarnos al servidor, crear bases de datos y tablas, escribir consultas SQL, ejecutar comandos, ver resultados y administrar objetos de la base.
En la práctica, SSMS va a ser nuestro entorno de trabajo: allí vamos a escribir SELECT, INSERT, UPDATE y DELETE, crear estructuras y probar lo que vayamos viendo en clase.

SQL Server Management Studio (SSMS)

SQL Server Management Studio (SSMS) es un entorno integrado gratuito de Microsoft para administrar, configurar y desarrollar cualquier infraestructura de SQL, desde SQL Server hasta Azure SQL Database. Proporciona herramientas gráficas y editores de scripts para gestionar bases de datos, ejecutar consultas T-SQL y automatizar tareas.

SSMS es una aplicación independiente del motor de base de datos y se debe descargar e instalar por separado. Es compatible con Windows y se actualiza periódicamente para mejorar la compatibilidad con las últimas características de SQL Server.
  • 1
    Gestión Integral: Permite administrar, configurar y desarrollar componentes del motor de base de datos SQL Server y Azure SQL.
  • 2
    Interfaz Gráfica (GUI): Incluye un Explorador de Objetos para visualizar y gestionar bases de datos, tablas y demas objetos de una base de datos.
  • 3
    Editor de Consultas: Potente editor para escribir, depurar y ejecutar codigo Transact-SQL (T-SQL).
  • 4
    Integraciones Modernas (versiones recientes): Las versiones nuevas (como SSMS 21) incluyen temas oscuros, integraciones con Git para control de versiones y soporte para IA a través de Azure Open AI.
  • 5
    Conectividad: Se conecta a instancias locales de SQL Server, SQL Server en máquinas virtuales de Azure y Azure SQL Database.
  • 6
    Automatización: Permite guardar scripts para ejecuciones posteriores y generar informes de configuración.
